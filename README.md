# 校園活動相簿管理系統

**Campus Activity Albums ‧ 新竹市內湖國小**

一套校園活動相簿系統。各處室同仁用 Google 帳號登入後台、建立活動相簿、上傳照片；
家長端是一個正式的官方網頁，可瀏覽相簿、依學年度與處室篩選、搜尋、下載照片。

---

## 系統架構

整套系統由四個免費服務組成，沒有自建伺服器，年度雲端費用為零。

```
   家長（免登入）              處室同仁（Google 登入）
        │                              │
        ▼                              ▼
  ┌───────────┐              ┌───────────────┐
  │ 家長端首頁 │              │ 後台管理介面   │
  │ index.html│              │後台管理介面.html│
  └─────┬─────┘              └───────┬───────┘
        │   讀取相簿／照片            │  建立・編輯・發布
        └──────────────┬─────────────┘
                       ▼
            ┌──────────────────────┐
            │  GAS 後端（Web App）   │  ← gas/ 資料夾內的程式
            │  驗證登入・讀寫資料     │
            └──────────┬───────────┘
                       ▼
        ┌──────────────────────────────┐
        │ Google Sheets（相簿／使用者資料）│
        │ Google Drive（照片檔案）        │
        └──────────────────────────────┘
```

- **前端**（`index.html`、`後台管理介面.html`）：靜態網頁，放在 GitHub Pages。
- **後端**（`gas/` 內的 `.gs`）：Google Apps Script，負責登入驗證與資料讀寫。
- **資料庫**：Google 試算表「校園活動相簿_資料庫」，含 Albums／Users／Config 三張表。
- **照片倉庫**：Google Drive「校園活動相簿」資料夾，一場活動一個子資料夾。

## 常見維護工作

### 新增一位後台同仁

- **學校 `@nhps.hc.edu.tw` 帳號**：不必登記，請對方直接用後台網址登入即可。
- **個人 Gmail 的同仁**：管理員登入後台 →「👥 使用者管理」→ 填 email、姓名、
  角色、處室 →「加入名單」（系統會自動開啟 Drive 上傳權限）。
- **設定管理員**：同樣在「使用者管理」頁，角色選「管理員」。

> 前提：Google 登入的 OAuth 同意畫面須已「發布為正式版」，否則僅限測試使用者
> 能登入。發布為一次性設定，基本登入不需 Google 審核。

### 更新網站內容（改了 html 後）

到 GitHub 儲存庫 →「Add file」→「Upload files」→ 拖入新檔案覆蓋 → Commit。

### 改了 GAS 程式後

Apps Script →「部署」→「管理部署作業」→ 編輯 → 版本選「**新版本**」→ 部署。

### 相簿資料在哪、出問題怎麼查

- 相簿清單、使用者名單：資料庫試算表（緊急時可直接在試算表改）。
- 照片檔案：Drive「校園活動相簿」資料夾。
- 相簿不見了：多半是 `status` 還在「草稿」，到後台發布即可。

## 交接須知

這套系統目前由一人維護，本 README 與各份部署說明即為交接文件。接手者請注意：

- **帳號歸屬**：系統目前建立在個人 Gmail 帳號下。長期建議把 Apps Script 專案、
  資料庫試算表、Drive 資料夾移轉到學校 `@nhps.hc.edu.tw` 帳號，讓這份校園影像
  檔案庫掛在會隨職務交接延續的穩定帳號上。
- **肖像權**：學生影像上公開網站，須有家長影像使用同意書為據。後台發布前有確認
  勾選；家長下載的照片帶校名浮水印、解析度已縮小。詳見規劃文件第九節。
- **重要設定值**：OAuth 用戶端 ID 在 `gas/01_Config.gs` 與 `後台管理介面.html`；
  後端網址、資料庫 ID、Drive 資料夾 ID 在 Apps Script 的「指令碼屬性」。
